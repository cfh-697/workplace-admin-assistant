# Workplace-Admin-Assistant | 理财部本地微型行政助手 (Qwen-MoE 轻量安全版)

> **本项目源自广东南海农商银行理财业务部行政助理实习期间的跨部门协作提效实践。我们设计并实现了一套基于本地局域网隔离运行的智能行政 Q&A 问答与 Qwen-MoE（Mixture of Experts）专家路由追踪系统。**

---

## 📌 项目背景与安全合规

在农商行理财部等高度保密金融环境下：
1.  **数据隐私红线**：内部行政规章、印章申请流向、员工个人隐私数据均严禁通过公网大模型（如 ChatGPT）转译，否则触犯金融安全红线。
2.  **本地算力瓶颈**：银行大楼内部服务器显存和算力有限，部署完整的大参数量大模型（如 70B）成本高昂。

---

## 💡 Qwen-MoE（混合专家模型）架构提效方案

混合专家模型（MoE）是解决本地算力约束与隐私合规的完美方案：
*   **计算公式**：
    $$\text{Output} = \sum_{i=1}^{N} G(x)_i E_i(x)$$
    其中，门控网络 $G(x)_i$ 根据输入提问向量计算出权重分配，**仅激活前 $k$ 个专家网络（$E_i(x)$）进行实质性计算**，其余专家保持静默，从而极大地降低了 GPU 显存消耗。
*   **本地物理隔离**：基于 **Ollama** 离线调用本地部署的 `qwen2.5-moe:1.5b`，让所有提问与行政策略只在本地网络流转，杜绝一切外泄风险。
*   **组织协作有温度**：结合大模型的 prompt engineering，克服了传统 FAQ 知识库死板的缺点，输出语气体面得体，彰显部门助理的关怀温度。

---

## 🖥️ 系统功能模块与可视化

本系统采用 **Python + Streamlit** 开发了极简、精美的交互式面板：

1.  **💬 智能问答大厅 (Chat)**：员工在此进行自然语言提问（如“卡纸了怎么办”），AI 自动匹配本地规章并输出极具人文关怀的回答。
2.  **📖 知识库索引 (Index)**：直观折叠展示 IT、财务、会议室、物料等不同分类下的标准行政管理规范。
3.  **🧠 MoE 专家路由追踪可视化 (Expert Routing)**：
    *   **核心特色**：系统后台通过语义词库拦截器分析，**在前端可视化模拟出门控网络（Gating Network）的分配比率**！
    *   例如当问及“报销”时，系统会在前端显示 **“财务报销与资产审计专家”** 处于 **🔥 激活 (Active) (90.0%)** 状态，其余专家处于挂起状态，直观科普 MoE 的运作逻辑。

## 🚀 运行指南 (Getting Started)

### 1. 启动本地 Ollama 服务
拉取轻量化混合专家模型：
```bash
ollama run qwen2.5-moe:1.5b
```

### 2. 启动智能行政面板
进入工程根目录并安装依赖，然后在本地 `8503` 端口运行面板：
```bash
pip install streamlit numpy
streamlit run app.py --server.port 8503 --server.headless true
```
打开浏览器访问 [http://localhost:8503](http://localhost:8503) 即可体验。
